# 以太坊提案研究员 (Ethereum Proposal Researcher)

该技能专注于分析前沿的以太坊改进提案（EIP）和研究性讨论。

## 使用场景 (When to Use)
- **技术预研**: 用户询问“某个提案（如 EIP-4844）会对现有的 L2 产生什么影响？”
- **路线图解析**: 理解以太坊未来 6-12 个月的发展方向。
- **债务评估**: 评估采用新技术对项目带来的迁移成本（技术债）。
- **可行性论证**: 分析某个研究方案是否具备在主网落地的前提。

## 边界与衔接 (Boundaries)
- **如果提案涉及具体的 DeFi 合约实现** -> 移交给 `ethereum-defi-analyzer`。
- **如果需要验证该提案在链上的早期实验数据** -> 移交给 `ethereum-chain-explorer`。

## 核心工作流 (Workflow)
1. **内容提取**: 使用 `utils/proposal_fetcher.py` 抓取 `ethresear.ch` 提案全文。
2. **分类识别**: 归类为共识层、执行层、ERC 或应用层提案。
3. **分层分析**: 评估对 EVM、网络层、智能合约兼容性的具体影响。
4. **债务核算**: 估算迁移该技术所需的开发、测试和运维工时。
5. **路线图定位**: 确定该提案目前处于哪个阶段（讨论中、草案、审核中、已激活）。

## 输出格式
- **核心摘要**: 提案解决的问题及主要手段。
- **技术分值 (1-5)**: 技术可行性、价值主张、风险等级。
- **技术债量化**: 迁移成本估算。
- **最终建议**: 采用/观望/拒绝。
